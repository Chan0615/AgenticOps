# 手动部署指南

在 Linux 服务器上手动部署 AgenticOps。

## 环境要求

| 依赖 | 版本 | 用途 |
|------|------|------|
| Python | 3.10+ | 后端运行 |
| Node.js | 18+ | 前端构建 |
| MySQL | 8.0+ | 主数据库 |
| Redis | 6+ | 任务队列 + 缓存 |
| PostgreSQL + pgvector | 16+ | RAG 向量数据库 |
| Nginx | 最新 | 反向代理 |

## 一、安装系统依赖

### Ubuntu / Debian

```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-pip \
  nodejs npm nginx git mysql-client redis-tools
```

### CentOS / RHEL

```bash
sudo yum install -y python3 python3-pip \
  nodejs npm nginx git redis
```

## 二、部署后端

```bash
# 1. 克隆代码
cd /opt
git clone <your-repo-url> AgenticOps
cd AgenticOps/backend

# 2. 创建虚拟环境
python3 -m venv .venv
source .venv/bin/activate

# 3. 安装依赖
pip install -r requirements.txt
pip install PyPDF2 python-docx openpyxl asyncpg pgvector

# 4. 配置文件
cp config.yaml.example config.yaml
# 编辑 config.yaml，填入实际的数据库、Redis、AI 等配置
vi config.yaml

# 5. 初始化数据库
python init_db.py

# 6. 初始化 pgvector 向量表
python init_pgvector.py

# 7. 测试启动
uvicorn app.main:app --host 0.0.0.0 --port 8000
# 访问 http://<server-ip>:8000/health 确认正常后 Ctrl+C 退出
```

## 三、部署前端

```bash
cd /opt/AgenticOps/frontend

# 安装依赖
npm install

# 构建生产版本
npm run build
# 产物在 dist/ 目录
```

## 四、配置 systemd 服务

创建 3 个 systemd 服务文件：

### API 服务

```bash
sudo tee /etc/systemd/system/agenticops-api.service << 'EOF'
[Unit]
Description=AgenticOps API Server
After=network.target mysql.service redis.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/AgenticOps/backend
ExecStart=/opt/AgenticOps/backend/.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
Restart=always
RestartSec=5
Environment=PATH=/opt/AgenticOps/backend/.venv/bin:/usr/bin

[Install]
WantedBy=multi-user.target
EOF
```

### Celery Worker

```bash
sudo tee /etc/systemd/system/agenticops-worker.service << 'EOF'
[Unit]
Description=AgenticOps Celery Worker
After=network.target redis.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/AgenticOps/backend
ExecStart=/opt/AgenticOps/backend/.venv/bin/celery -A celery_app worker --loglevel=info -Q salt,scheduler
Restart=always
RestartSec=5
Environment=PATH=/opt/AgenticOps/backend/.venv/bin:/usr/bin

[Install]
WantedBy=multi-user.target
EOF
```

### Celery Beat

```bash
sudo tee /etc/systemd/system/agenticops-beat.service << 'EOF'
[Unit]
Description=AgenticOps Celery Beat
After=network.target redis.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/AgenticOps/backend
ExecStart=/opt/AgenticOps/backend/.venv/bin/celery -A celery_app beat --loglevel=info
Restart=always
RestartSec=5
Environment=PATH=/opt/AgenticOps/backend/.venv/bin:/usr/bin

[Install]
WantedBy=multi-user.target
EOF
```

### 启动服务

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now agenticops-api agenticops-worker agenticops-beat

# 检查状态
sudo systemctl status agenticops-api
sudo systemctl status agenticops-worker
sudo systemctl status agenticops-beat
```

## 五、配置 Nginx

```bash
sudo tee /etc/nginx/sites-available/agenticops << 'EOF'
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    location / {
        root /opt/AgenticOps/frontend/dist;
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff2?)$ {
        root /opt/AgenticOps/frontend/dist;
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

        # SSE 支持
        proxy_buffering off;
        proxy_cache off;
        proxy_http_version 1.1;
        proxy_set_header Connection "";
        proxy_read_timeout 3600s;
        proxy_send_timeout 3600s;
    }

    # 文件上传大小限制
    client_max_body_size 50m;
}
EOF

# 启用站点
sudo ln -sf /etc/nginx/sites-available/agenticops /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## 六、更新部署

```bash
cd /opt/AgenticOps

# 拉取最新代码
git pull

# 后端更新
cd backend
source .venv/bin/activate
pip install -r requirements.txt

# 运行增量迁移脚本（如果有新模块）
# python add_dataquery_menu.py
# python init_pgvector.py

sudo systemctl restart agenticops-api agenticops-worker agenticops-beat

# 前端更新
cd ../frontend
npm install
npm run build

# Nginx 会自动服务新的 dist/ 文件，无需重启
```

## 七、日志查看

```bash
# API 日志
sudo journalctl -u agenticops-api -f

# Worker 日志
sudo journalctl -u agenticops-worker -f

# Beat 日志
sudo journalctl -u agenticops-beat -f

# Nginx 日志
sudo tail -f /var/log/nginx/access.log
sudo tail -f /var/log/nginx/error.log
```

## 八、防火墙

```bash
# 开放 80/443 端口
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 如果需要外部直接访问 API（非推荐）
# sudo ufw allow 8000/tcp
```

## 九、HTTPS（可选）

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 自动配置 SSL
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run
```
