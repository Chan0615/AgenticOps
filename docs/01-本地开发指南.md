# 本地开发指南

本文档说明如何在本地 Windows / macOS / Linux 开发环境中启动 AgenticOps。

## 环境要求

| 依赖 | 版本 | 说明 |
|------|------|------|
| Python | 3.10+ | 推荐 3.12 |
| Node.js | 18+ | 推荐 20 LTS |
| MySQL | 8.0+ | 主数据库 |
| Redis | 6+ | Celery 队列 + 缓存 |
| PostgreSQL + pgvector | 16+ | RAG 向量数据库（可选，详见 [PGVECTOR_INSTALL.md](PGVECTOR_INSTALL.md)） |

## 一、首次配置

### 1.1 后端配置

```bash
cd backend

# 复制配置文件
cp config.yaml.example config.yaml

# 编辑 config.yaml，填入实际值：
#   - mysql: 数据库连接
#   - redis: Redis 连接
#   - pgvector: 向量数据库连接（RAG 功能必需）
#   - ai: AI 模型 API Key（至少启用一个：qwen / deepseek / openai）
#   - jwt.secret_key: 改为随机字符串
```

### 1.2 安装依赖

```bash
# 后端
cd backend
python -m venv .venv

# Windows
.venv\Scripts\activate
# Linux/macOS
source .venv/bin/activate

pip install -r requirements.txt

# 文档解析依赖（PDF/Word）
pip install PyPDF2 python-docx openpyxl

# pgvector 连接（RAG 功能）
pip install asyncpg pgvector

# 前端
cd frontend
npm install
```

### 1.3 初始化数据库

```bash
cd backend

# 初始化 MySQL 表 + 种子数据（管理员/角色/菜单）
# ⚠️ 警告：force_reset=True 会清空所有表！
python init_db.py

# 初始化 pgvector 向量表（幂等，可重复运行）
python init_pgvector.py
```

> 默认管理员账号：`admin / admin123`

### 1.4 增量迁移（已有数据库）

如果是已有数据库，不要跑 `init_db.py`，用增量脚本：

```bash
# 添加智能问数模块的菜单和表
python add_dataquery_menu.py

# 迁移旧版服务器菜单路径
python add_server_menu.py
```

## 二、启动服务

需要 4 个终端窗口（或使用终端复用工具）：

### 终端 1：后端 API

```bash
cd backend
.venv\Scripts\activate      # Windows
uvicorn app.main:app --reload --port 8000
```

### 终端 2：Celery Worker

```bash
cd backend
.venv\Scripts\activate

# Windows（必须加 -P solo）
celery -A celery_app worker --loglevel=info -Q salt,scheduler -P solo

# Linux/macOS
celery -A celery_app worker --loglevel=info -Q salt,scheduler
```

### 终端 3：Celery Beat

```bash
cd backend
.venv\Scripts\activate
celery -A celery_app beat --loglevel=info
```

### 终端 4：前端

```bash
cd frontend
npm run dev
```

## 四、常用命令

### 前端

```bash
npm run dev        # 开发模式（热重载）
npm run build      # 生产构建
npm run typecheck  # TypeScript 类型检查
npm run preview    # 预览构建产物
```

### 后端

```bash
python init_db.py               # 初始化/重建数据库（⚠️ 破坏性）
python init_pgvector.py         # 初始化 pgvector 向量表
python test_pgvector.py         # 测试 pgvector 连接
python add_dataquery_menu.py    # 智能问数菜单迁移（增量，安全）
python add_server_menu.py       # 运维菜单迁移（增量，安全）
python mcp_server.py            # 启动 MCP Server
```

## 五、注意事项

1. **config.yaml 是 gitignored 的**，不要提交敏感信息
2. **init_db.py 是破坏性的**（`force_reset=True`），会清空所有数据
3. **Windows Celery 必须加 `-P solo`**，否则会报错
4. **Ant Design Vue 未全局注册**，组件必须在每个文件中单独 import
5. **前端 Vite 代理** 将 `/api` 转发到 `localhost:8000`，后端必须在 8000 端口
6. **pgvector 是 RAG 功能的必需依赖**，不配置则知识库检索不可用（但系统仍可启动）
